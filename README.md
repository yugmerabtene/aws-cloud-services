### **IaaS (Infrastructure as a Service)**

1. **EC2 (Elastic Compute Cloud)**  
   - **Signification** : Service de calcul à la demande.  
   - **Explication** : Machines virtuelles pour exécuter des applications.

2. **S3 (Simple Storage Service)**  
   - **Signification** : Stockage d'objets.  
   - **Explication** : Stockage sécurisé et durable pour les données volumineuses.

3. **EBS (Elastic Block Store)**  
   - **Signification** : Stockage en bloc élastique.  
   - **Explication** : Disques durs virtuels pour instances EC2.

4. **VPC (Virtual Private Cloud)**  
   - **Signification** : Cloud privé virtuel.  
   - **Explication** : Réseau isolé pour héberger des ressources cloud.

5. **RDS (Relational Database Service)**  
   - **Signification** : Service de base de données relationnelle.  
   - **Explication** : Gère des bases de données comme MySQL, PostgreSQL sans gestion de serveur.

### **PaaS (Platform as a Service)**

1. **Elastic Beanstalk**  
   - **Signification** : Déploiement automatisé.  
   - **Explication** : Déploiement d'applications sans gérer l'infrastructure.

2. **Lambda**  
   - **Signification** : Calcul sans serveur.  
   - **Explication** : Exécution de code sans gestion de serveur, facturation basée sur l'exécution.

3. **Fargate**  
   - **Signification** : Calcul sans serveur pour conteneurs.  
   - **Explication** : Exécute des conteneurs sans gérer l'infrastructure sous-jacente.

### **SaaS (Software as a Service)**

1. **WorkMail**  
   - **Signification** : Service de messagerie professionnelle.  
   - **Explication** : Hébergement d'emails et calendriers pour entreprises.

2. **Chime**  
   - **Signification** : Service de communication unifiée.  
   - **Explication** : Messagerie, appels vidéo et conférences pour entreprises.

3. **WorkDocs**  
   - **Signification** : Stockage et collaboration de documents.  
   - **Explication** : Partage et collaboration sur des documents en ligne.

### **IaC (Infrastructure as Code)**

1. **CloudFormation**  
   - **Signification** : Provisionnement d'infrastructure.  
   - **Explication** : Gestion de l'infrastructure via des fichiers de configuration.

2. **CDK (Cloud Development Kit)**  
   - **Signification** : Kit de développement d'infrastructure.  
   - **Explication** : Définir des ressources AWS avec des langages de programmation.

3. **OpsWorks**  
   - **Signification** : Gestion de configuration.  
   - **Explication** : Automatisation de la gestion de la configuration avec Chef et Puppet.

### **LaaS (Logging as a Service)**

1. **CloudWatch**  
   - **Signification** : Service de surveillance et de logs.  
   - **Explication** : Surveillance des métriques, journaux et événements AWS.

2. **AWS X-Ray**  
   - **Signification** : Analyse de performance.  
   - **Explication** : Suivi des requêtes et analyse des performances des applications.

### **BaaS (Backend as a Service)**

1. **AWS Amplify**  
   - **Signification** : Développement d’applications web et mobiles.  
   - **Explication** : Backend complet pour des applications, incluant API, authentification et stockage.

2. **AWS Cognito**  
   - **Signification** : Gestion des utilisateurs.  
   - **Explication** : Authentification et autorisation des utilisateurs pour les applications.

3. **AWS AppSync**  
   - **Signification** : API GraphQL managée.  
   - **Explication** : Synchronisation des données entre le backend et les applications via GraphQL.

### **CaaS (Container as a Service)**

1. **ECS (Elastic Container Service)**  
   - **Signification** : Service de gestion de conteneurs.  
   - **Explication** : Gestion et exécution de conteneurs Docker.

2. **EKS (Elastic Kubernetes Service)**  
   - **Signification** : Service de Kubernetes managé.  
   - **Explication** : Orchestration de conteneurs via Kubernetes.

3. **Fargate**  
   - **Signification** : Exécution de conteneurs sans gestion de serveurs.  
   - **Explication** : Permet d'exécuter des conteneurs sans gérer les serveurs.

### **FaaS (Function as a Service)**

1. **Lambda**  
   - **Signification** : Calcul sans serveur.  
   - **Explication** : Exécution de fonctions sur demande sans gestion de serveur.

2. **Step Functions**  
   - **Signification** : Orchestration de services.  
   - **Explication** : Coordination de fonctions Lambda et autres services AWS avec des workflows.

3. **API Gateway**  
   - **Signification** : Passerelle API.  
   - **Explication** : Créer et gérer des API RESTful ou WebSocket.

### **DaaS (Data as a Service)**

1. **Amazon Redshift**  
   - **Signification** : Entrepôt de données cloud.  
   - **Explication** : Analyse de données à grande échelle.

2. **Amazon Athena**  
   - **Signification** : Analyse SQL interactive.  
   - **Explication** : Requêtes SQL sur des données dans S3.

3. **AWS Glue**  
   - **Signification** : Service ETL (Extraction, Transformation, Chargement).  
   - **Explication** : Préparation et transformation des données pour les analyses.

### **MBaaS (Mobile Backend as a Service)**

1. **AWS Amplify**  
   - **Signification** : Développement d'applications mobiles.  
   - **Explication** : Backend complet pour le développement d'applications mobiles.

2. **Amazon Pinpoint**  
   - **Signification** : Gestion de campagnes marketing.  
   - **Explication** : Notifications push, emails et SMS pour applications mobiles.

### **MaaS (Monitoring as a Service)**

1. **AWS CloudTrail**  
   - **Signification** : Audit et surveillance.  
   - **Explication** : Enregistre toutes les actions API AWS pour surveillance et audit.

2. **AWS Config**  
   - **Signification** : Surveillance de la configuration.  
   - **Explication** : Suivi et gestion des configurations des ressources AWS.

### **DBaaS (Database as a Service)**

1. **Amazon Aurora**  
   - **Signification** : Base de données compatible MySQL et PostgreSQL.  
   - **Explication** : Base de données relationnelle hautement performante.

2. **Amazon DynamoDB**  
   - **Signification** : Base de données NoSQL.  
   - **Explication** : Base de données rapide pour des applications nécessitant des temps de réponse rapides.

3. **Amazon Neptune**  
   - **Signification** : Base de données orientée graphe.  
   - **Explication** : Gestion des données complexes sous forme de graphe.

### **MLaaS (Machine Learning as a Service)**

1. **SageMaker**  
   - **Signification** : Plateforme de machine learning.  
   - **Explication** : Entraînement, déploiement et gestion des modèles de machine learning.

2. **Amazon Rekognition**  
   - **Signification** : Reconnaissance d'images et vidéos.  
   - **Explication** : Analyse d'images et de vidéos pour détecter objets et visages.

3. **Amazon Polly**  
   - **Signification** : Conversion texte en parole.  
   - **Explication** : Convertit du texte en voix synthétisée.

4. **Amazon Lex**  
   - **Signification** : Service de chatbot.  
   - **Explication** : Création de chatbots avec compréhension du langage naturel.

### **XaaS (Anything as a Service)**

1. **Amazon QLDB**  
   - **Signification** : Base de données de registre.  
   - **Explication** : Ledger distribué, immuable et transparent pour des transactions vérifiables.

2. **Amazon Sumerian**  
   - **Signification** : Création de contenu AR/VR.  
   - **Explication** : Création d’expériences de réalité virtuelle et augmentée.

### **DRaaS (Disaster Recovery as a Service)**

1. **AWS Backup**  
   - **Signification** : Gestion de sauvegarde.  
   - **Explication** : Sauvegarde et restauration centralisées des ressources AWS.

2. **AWS Elastic Disaster Recovery (

DRS)**  
   - **Signification** : Récupération après sinistre.  
   - **Explication** : Récupération rapide des environnements en cas de sinistre.
